CB FORGE - STATO LAVORI
Data aggiornamento: 2026-03-15 sera

OBIETTIVO REALE DEL PROGETTO
- Costruire un tool che legga i dati locali di RAID Shadow Legends.
- Registrare fight reali.
- Estrarre dati utili per capire come migliorare team e campioni.
- In futuro normalizzare tutto in database e poi usare analytics/AI per dare suggerimenti.

STATO ATTUALE - COSA FUNZIONA DAVVERO
- Server web funzionante su:
  - http://127.0.0.1:8765
- Registrazione run da interfaccia web funzionante.
- Feed live del combat funzionante.
- Lettura team dal log di RAID funzionante.
- Cattura automatica del file:
  - battle-results/battleResults
- Watcher live del battleResults funzionante anche in background.
- Decodifica del battleResults ora fatta principalmente in Python:
  - msgpack
  - lz4
- Salvataggio capture in:
  - input/battle_result_captures
- Salvataggio storico run in:
  - input/cb_manual_runs.jsonl
- Recovery automatico se la sessione attiva sparisce prima dello stop:
  - source = recovered_session

ULTIMA VERIFICA POSITIVA
- L'ultima run e' stata salvata davvero nello storico.
- E' presente il capture nuovo:
  - .bin
  - .json
  - .meta.json
- La run contiene anche battle_result_capture e damage_summary.

DATI CHE RIUSCIAMO GIA' A PRENDERE
- team usato nel fight
- battle_id
- stato del combat
- log live client/sqlite
- damage totale stimato della run
- damage per champion stimato

NOTA IMPORTANTE SUL DANNO
- Il damage per champion e' attualmente "best effort" / "heuristic".
- Nei payload battleResults non sempre esiste lo stesso campo.
- Priorita' attuale del parser:
  1. ad.2004
  2. h
  3. dt
  4. s
- I numeri sono utili per confronto tra run, ma non sono ancora da considerare "verita' assoluta" al 100%.
- AGGIUNTA IMPORTANTE 2026-03-15:
  - Per il Clan Boss il damage automatico attuale NON e' affidabile.
  - In test reale due run che in game valevano insieme quasi 70M sono state lette come circa 1.8M totali.
  - Quindi per ora il campo damage letto dal battleResults per il Clan Boss va considerato debug, non dato analytics.
  - Le run restano comunque valide per:
    - team usato
    - capture raw
    - battle_id
    - snapshot
    - eventi live
  - Quando serve analisi seria del Clan Boss, usare il danno reale manuale finche' non viene trovato il campo giusto nel payload.

Il prossimo lavoro utile NON e' sul feed base, ma sulla fase successiva:

1. NORMALIZZAZIONE DATI RUN
- Creare un piccolo database SQLite.

2. OBIETTIVO ANALYTICS PRIMA DELL'AI
- Confrontare run tra loro.
- Capire:
  - chi fa meno danno
  - chi regge peggio
  - quali team sono piu' stabili
  - quali combinazioni migliorano davvero
- Prima fare report rule-based.
- Solo dopo pensare a una AI vera.

3. DATI CHE MANCANO ANCORA PER FARE ANALISI SERIA
- skill per champion durante il fight
- buff/debuff applicati
- danni ricevuti
- ordine morti
- boss skill -> target -> effetto

Questi dati NON sono ancora normalizzati bene.
Per arrivarci bisogna estendere il tracciamento eventi, non inseguire subito il modello AI.
